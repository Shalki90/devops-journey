# Purpose

This document defines the **current official version state of the repository**.

It provides a single source of truth for understanding the maturity and lifecycle stage of the DevOps Engineering Journey repository.

---

# Repository Versioning Philosophy

The repository follows a **hybrid versioning model**:

* The **repository itself** has a global version.
* Each **document within the repository** has its own independent version (starting at 1.0).
* Each **sprint represents a structured evolution milestone**, not a version increment.

This ensures clarity between:

* Repository-level evolution
* Document-level evolution
* Sprint-based progress

# Version Meaning

| Version | Meaning                                       |
| ------- | --------------------------------------------- |
| 0.x     | Initial setup / unstable structure            |
| 1.0     | Stable architecture + migration in progress   |
| 1.x     | Incremental improvements to structure/content |
| 2.0     | Major structural redesign or paradigm shift   |

---

# Version Scope

Repository version applies to:

* Overall architecture
* Governance system
* Documentation model
* Knowledge organization strategy

It does NOT apply to:

* Individual concept files
* Technology-specific documentation
* Labs or projects

Those follow independent Version 1.0+ lifecycle per file.

---

# Release Criteria for 1.0 Completion

The repository will be considered fully Version 1.0 complete when:

* All planned sprints are completed.
* All knowledge from the source repository is migrated.
* No duplicate or orphaned content remains.
* All documents follow governance rules.
* Navigation is consistent and predictable.
* README is finalized and aligned with structure.

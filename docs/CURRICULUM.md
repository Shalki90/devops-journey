# Curriculum

## Purpose

This curriculum defines the learning path for the Engineering OS.

Each phase is maintained as an independent document to keep the curriculum modular, scalable, and easy to navigate.

The curriculum may evolve as better learning sequences are discovered.

## Design Principles

- Each phase is maintained as an independent document.
- Every phase follows a standard template.
- Phase documents focus on concepts, learning outcomes, and practical understanding rather than session notes.
- Supporting documents (Dictionary, Analogies, Engineering Principles, AI in Practice, and Questions Parked) evolve alongside the curriculum.
- The learning sequence may evolve when a different order provides stronger conceptual foundations.

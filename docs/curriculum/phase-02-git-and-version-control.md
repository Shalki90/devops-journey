# Phase 02 — Git & Version Control

## Objective

Understand why version control exists, how Git manages change over time, and how modern engineering teams collaborate safely without overwriting each other's work.

The goal is to develop a mental model of Git as a change-management system rather than memorizing commands.

---

## Learning Outcomes

By the end of this phase, I should be able to:

* Explain why version control exists.
* Describe the relationship between Git and GitHub.
* Explain the purpose of repositories.
* Differentiate between Working Directory, Staging Area, Local Repository, and Remote Repository.
* Explain the complete Git commit lifecycle.
* Understand commit history and commit hashes.
* Create and use branches safely.
* Explain branch isolation.
* Perform merges.
* Explain fast-forward merges.
* Explain merge commits.
* Create and resolve merge conflicts.
* Understand remote tracking branches.
* Explain the difference between fetch, pull, and push.
* Explain how teams collaborate using branches.
* Understand GitHub Pull Request workflows.
* Explain when to use revert, reset, and stash.
* Reason about Git operations using real-world analogies.

## Hands-on Labs

| Status | Lab                         | Objective                                                  |
| ------ | --------------------------- | ---------------------------------------------------------- |
| ✅      | Initialize Local Repository | Create and manage a Git repository locally.                |
| ✅      | Create Commits              | Observe how changes move through the Git lifecycle.        |
| ✅      | Inspect History             | Use Git history commands to understand commit tracking.    |
| ✅      | Create Branches             | Demonstrate branch isolation and independent development.  |
| ✅      | Fast-Forward Merge          | Observe branch pointer movement without merge commits.     |
| ✅      | Merge Conflict Creation     | Intentionally create conflicting changes.                  |
| ✅      | Conflict Resolution         | Manually resolve merge conflicts and create merge commits. |
| ✅      | Push to GitHub              | Publish local commits to a remote repository.              |
| ✅      | Fetch Remote Changes        | Update remote references without modifying local branches. |
| ✅      | Pull Remote Changes         | Synchronize local branches with remote branches.           |
| ✅      | Revert vs Reset Exercises   | Compare published-history fixes versus history rewriting.  |
| ✅      | Stash Workflow Exercise     | Temporarily save unfinished work and restore it later.     |
| ✅      | Git Analogy Challenge       | Reinforce Git concepts using book-writing analogies.       |

## Engineering Principles Reinforced

During this phase, the following engineering principles were reinforced:

* History Is Valuable
* Change Tracking Enables Reliability
* Isolation Reduces Risk
* Safe Experimentation
* Review Before Integration
* Preserve Shared History
* Incremental Progress
* Recoverability
* Collaboration Through Process
* Visibility Enables Understanding

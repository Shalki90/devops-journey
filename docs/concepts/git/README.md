# Purpose

This document is the **entry point for Git and Version Control knowledge** in the DevOps Engineering Journey repository.

It consolidates all Git-related concepts into a structured learning module designed for both practical usage and interview readiness.

---

# What is Version Control

| Concept         | Meaning                                        |
| --------------- | ---------------------------------------------- |
| Version Control | System that tracks changes in files over time  |
| Source Control  | Managing source code changes collaboratively   |
| Repository      | Storage location for project files and history |

---

# Why Git Matters

| Reason           | Explanation                           |
| ---------------- | ------------------------------------- |
| Collaboration    | Multiple developers can work together |
| History tracking | Every change is recorded              |
| Rollback ability | Can revert to previous versions       |
| Branching        | Enables parallel development          |

---

# Git Ecosystem

| Component         | Role                               |
| ----------------- | ---------------------------------- |
| Working Directory | Local files being edited           |
| Staging Area      | Prepared changes before commit     |
| Local Repository  | Git history stored locally         |
| Remote Repository | Shared repository (GitHub, GitLab) |

---

# Core Git Operations

| Operation | Description                      |
| --------- | -------------------------------- |
| clone     | Copy remote repository locally   |
| add       | Stage changes                    |
| commit    | Save snapshot of changes         |
| push      | Send changes to remote           |
| pull      | Fetch and merge changes          |
| fetch     | Download changes without merging |

---

# Git Workflow Model

| Step | Action             |
| ---- | ------------------ |
| 1    | Modify files       |
| 2    | Stage changes      |
| 3    | Commit changes     |
| 4    | Push to remote     |
| 5    | Collaborate via PR |

---

# Branching Concept

| Concept        | Explanation                     |
| -------------- | ------------------------------- |
| Branch         | Independent line of development |
| Main branch    | Stable production-ready code    |
| Feature branch | Used for new features           |
| Merge          | Combine branches                |

---

# Key Insight

Git is not just a tool — it is a **time machine for code**.

| Capability    | Meaning              |
| ------------- | -------------------- |
| Snapshotting  | Save project state   |
| History       | Track evolution      |
| Collaboration | Parallel development |
| Recovery      | Undo mistakes        |

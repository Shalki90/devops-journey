**Document Version:** 1.0
**Sprint:** 1 — Internet & Networking

# Purpose

This document captures **mental models and analogies** for networking concepts extracted from the DevOps Engineering Journey repository.

It is designed to help convert abstract networking concepts into **real-world intuitive understanding**.

---

# Core Networking Analogies

| Concept       | Analogy                       | Explanation                                                                                       |
| ------------- | ----------------------------- | ------------------------------------------------------------------------------------------------- |
| Internet      | Road network between cities   | The Internet connects multiple networks just like roads connect cities globally.                  |
| Network       | Local city transport system   | A network connects devices within a limited area like a city transport system connects locations. |
| IP Address    | Home address                  | Identifies exactly where a device is located on a network.                                        |
| DNS           | Phone book                    | Translates human-readable domain names into IP addresses.                                         |
| Router        | Traffic police                | Directs data traffic to the correct destination path.                                             |
| Packet        | Letter in postal system       | Data is broken into small packets like letters sent through mail.                                 |
| Client        | Customer                      | Requests services or data.                                                                        |
| Server        | Service provider / kitchen    | Processes requests and returns responses.                                                         |
| Protocol      | Language rules                | Defines how communication happens between systems.                                                |
| Port          | Apartment number              | Identifies which application inside a system should receive data.                                 |
| Socket        | Full address (IP + Port)      | Combination that uniquely identifies a communication endpoint.                                    |
| HTTP          | Ordering system in restaurant | Standard way for clients and servers to communicate.                                              |
| HTTPS         | Secure sealed envelope        | HTTP with encryption for privacy and security.                                                    |
| Firewall      | Security guard at gate        | Controls what traffic is allowed in or out.                                                       |
| Load Balancer | Reception desk                | Distributes incoming requests evenly across servers.                                              |
| NAT           | Office receptionist           | Maps multiple internal users to a single external identity.                                       |
| Routing       | GPS navigation system         | Finds the best path for data to reach destination.                                                |
| TCP           | Phone call                    | Reliable connection with acknowledgment between two parties.                                      |
| UDP           | Radio broadcast               | Fast communication without confirmation of delivery.                                              |
| Encryption    | Locked safe                   | Data is protected so only authorized users can access it.                                         |
| Public Key    | Open padlock                  | Anyone can lock data using it.                                                                    |
| Private Key   | Unique key                    | Only the owner can unlock encrypted data.                                                         |

# Key Insight

Networking becomes significantly easier when mapped to **real-world systems**:

| Domain              | Mapping                       |
| ------------------- | ----------------------------- |
| Geography           | Internet, routing, DNS        |
| Postal system       | Packets, delivery, addressing |
| Human communication | Client-server, protocols      |
| Security systems    | Firewalls, encryption         |
| Transportation      | Routing, traffic control      |
